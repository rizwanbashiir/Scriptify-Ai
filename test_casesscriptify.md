# Backend End-to-End Test Cases

This document outlines the test cases you can use in Postman (or ThunderClient/Insomnia) to verify the entire end-to-end functionality of your backend.

## 1. User Management & Authentication

- [ ] **Sign Up (`POST /api/users/signup`)**
  - **Action:** Send `firstName`, `lastName`, `email`, `password`.
  - **Expected:** Returns `201 Created`, creates a user in DB, and sends an OTP to the email address.
- [ ] **Verify Email (`POST /api/users/verify-email`)**
  - **Action:** Send the `email` and the 6-digit `otp` received via email.
  - **Expected:** Returns `200 OK` and sets `isVerified` to true.
- [ ] **Sign In (`POST /api/users/signin`)**
  - **Action:** Send `email` and `password`.
  - **Expected:** Returns `200 OK` along with `accessToken` and `refreshToken`.
- [ ] **Forgot Password (`POST /api/users/forgot-password`)**
  - **Action:** Send `email`.
  - **Expected:** Returns `200 OK` and sends a reset OTP to the email.
- [ ] **Reset Password (`POST /api/users/reset-password`)**
  - **Action:** Send `email`, `otp`, and `newPassword`.
  - **Expected:** Returns `200 OK` and updates the password in the database.
- [ ] **Refresh Token (`POST /api/users/refresh-token`)**
  - **Action:** Send the `refreshToken`.
  - **Expected:** Returns a new pair of access and refresh tokens.

## 2. Blog Management

> **Note:** Ensure you include the `Authorization: Bearer <accessToken>` header for authenticated routes.

- [ ] **Create a Blog (`POST /api/blogs`)**
  - **Action:** Send `title` and `content` as form-data (optionally attach a `thumbnail` file).
  - **Expected:** Returns `201 Created`. If a thumbnail was attached, verify it was uploaded to Cloudinary.
- [ ] **Get All Blogs (`GET /api/blogs`)**
  - **Action:** Request with optional query params like `?page=1&limit=10&search=keyword`.
  - **Expected:** Returns a paginated array of published blogs.
- [ ] **Get Blog Details (`GET /api/blogs/:id`)**
  - **Action:** Request a specific blog ID.
  - **Expected:** Returns the full blog object and increments the `views` count by 1.
- [ ] **Update a Blog (`PUT /api/blogs/:id`)**
  - **Action:** Modify the `title`, `content`, or `status`.
  - **Expected:** Returns the updated blog object.
- [ ] **Delete a Blog (`DELETE /api/blogs/:id`)**
  - **Action:** Delete a blog you own.
  - **Expected:** Returns `200 OK` and removes the blog (and associated comments/Cloudinary images).

## 3. Social Interactions & Recommendations

- [ ] **Toggle Like (`POST /api/blogs/:id/like`)**
  - **Action:** Like a blog post.
  - **Expected:** Returns `200 OK` and toggles the like state (increments or decrements count).
- [ ] **Toggle Bookmark (`POST /api/users/bookmarks/:blogId`)**
  - **Action:** Bookmark a blog.
  - **Expected:** Returns `200 OK` and adds the blog to the user's bookmarks list.
- [ ] **Toggle Follow (`POST /api/users/:id/follow`)**
  - **Action:** Follow another user (author).
  - **Expected:** Returns `200 OK` and updates followers/following arrays.
- [ ] **Get Personalized Feed (`GET /api/ai/recommendations/feed`)**
  - **Action:** Fetch the feed.
  - **Expected:** Returns a ranked array of blogs based on previous interactions (likes, views).

## 4. AI Integrations (Groq, Gemini, Hugging Face)

> **Important:** Ensure `GROQ_API_KEY`, `GEMINI_API_KEY`, and `HUGGINGFACE_API_KEY` are all set in your `.env`.

- [ ] **Generate Blog Draft (`POST /api/ai/generate-draft`)**
  - **Action:** Send `{ "topic": "Future of AI", "wordCount": 500 }`.
  - **Expected:** Returns a structured JSON blog post generated by the **Groq API**.
- [ ] **Improve Content (`POST /api/ai/improve-content`)**
  - **Action:** Send a messy paragraph of text.
  - **Expected:** Returns grammatically corrected and polished text.
- [ ] **Generate SEO Tags (`POST /api/ai/generate-seo-tags`)**
  - **Action:** Send `{ "title": "My Blog", "content": "..." }`.
  - **Expected:** Returns JSON array of tags, keywords, and a meta description.
- [ ] **Generate Thumbnail (`POST /api/ai/generate-thumbnail`)**
  - **Action:** Send `{ "title": "Future of AI" }`.
  - **Expected:** Returns an image URL. Verifies the **Gemini Imagen 3 API** (or Pollinations fallback) works and uploads to Cloudinary.
- [ ] **Add a Comment (Sentiment Analysis) (`POST /api/blogs/:id/comments`)**
  - **Action:** Add a highly toxic/offensive comment to a blog post.
  - **Expected:** The Hugging Face API catches it. The comment is saved but `isApproved` is `false`, meaning it won't show up publicly.

## 5. Admin Capabilities

> **Note:** These endpoints require a user with `role: "admin"`.

- [ ] **Get Dashboard Stats (`GET /api/admin/dashboard`)**
  - **Expected:** Returns platform-wide stats (total users, total views, new blogs this week).
- [ ] **Get Flagged Comments (`GET /api/admin/comments/flagged`)**
  - **Expected:** Returns a list of comments that the AI marked as `TOXIC`.
- [ ] **Moderate Comment (`PATCH /api/admin/comments/:commentId/moderate`)**
  - **Action:** Send `{ "action": "approve" }` or `{ "action": "delete" }`.
  - **Expected:** Applies the moderation decision to the flagged comment.
- [ ] **Toggle User Suspension (`PATCH /api/admin/users/:id/suspend`)**
  - **Action:** Suspend a user.
  - **Expected:** The suspended user can no longer log in.
